Guia prático de seletores CSS

Exemplos visuais para aula introdutória

1. O problema inicial

Em CSS, os seletores indicam exatamente quais elementos receberão um estilo.

Em outras palavras, eles definem onde cada regra será aplicada dentro da página. 

p {
  color: blue;
}

2. Seletor universal e por tipo

O seletor por tipo aplica estilo a todos os elementos de uma mesma tag.

Já o seletor universal afeta todos os elementos do documento, sendo útil para configurações gerais. 

* {
  box-sizing: border-box;
}

p {
  color: #334155;
  line-height: 1.6;
}

3. Classe e ID

Classes podem ser utilizadas em vários elementos.

Isso permite reutilizar estilos e manter consistência visual. 

.destaque {
  background: #eff6ff;
  border-left: 4px solid #2563eb;
  font-weight: bold;
}

IDs devem ser únicos dentro da página.

Eles são usados para identificar elementos específicos. 

#aviso-unico {
  color: #8b0000;
  font-style: italic;
}

O seletor pode atingir elementos dentro de outros, como links dentro de um menu. 

nav a {
  text-decoration: none;

Também é possível selecionar apenas elementos filhos diretos. 

ul.menu-filho > li {
  background: #f8fafc;
  padding: 6px;
}

5. Pseudo-classes

Passe o mouse aqui

Pseudo-classes representam estados como interação do usuário. 

.botao:hover {
  background: #1d4ed8;
  color: white;
}

input:focus {
  outline: 2px solid #2563eb;
}

Algumas pseudo-classes selecionam elementos pela posição. 

.lista-zebrada li:nth-child(2n) {
  background: #e0f2fe;
}

6. Pseudo-elementos

Texto com conteúdo antes e depois

Permitem adicionar conteúdo visual sem alterar o HTML. 

.pseudo::before {
  content: ">> ";
}

.pseudo::after {
  content: " <<";
}

Este texto destaca a primeira letra. 

.primeira-letra::first-letter {
  font-size: 2rem;
  font-weight: bold;
  color: #0b51bb;
}

7. Especificidade

Este elemento recebe estilos de vários seletores

Quando há conflito entre regras, o CSS aplica a mais específica.

Em geral, IDs têm prioridade maior que classes e seletores simples. 

p {
  color: #334155;
}

.destaque {
  color: #ea580c;
}

#principal {
  color: #b91c1c;
}

8. Resumo

  • Seletores definem quais elementos recebem estilo
  • Classes podem ser reutilizadas
  • IDs devem ser únicos
  • Combinadores dependem da estrutura
  • Pseudo-classes representam estados
  • Pseudo-elementos criam conteúdo visual
  • Especificidade define prioridade
